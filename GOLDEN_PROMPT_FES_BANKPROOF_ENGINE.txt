--------------------------------------------------------------------
Implement ONE engine with TWO strictly separated layers:

LAYER A – Business‑Model Economic Layer (heterogeneous)
- BM‑specific revenues
- BM‑specific costs
- BM‑specific stochastic drivers

LAYER B – Financial Risk & Bankability Layer (homogeneous)
- identical risk metrics for all BMs
- identical bankability logic
- stress‑scenario dominance

Never merge these layers.

--------------------------------------------------------------------
BUSINESS MODEL AS CONFIGURATION (NO HARD‑CODING)
--------------------------------------------------------------------
Each business model MUST be defined via a configuration object
(YAML / JSON / Python dict), not ad‑hoc logic.

Example structure:

business_model:
  id: FR
  name: Recreational & Water Services
  duration_years: 20
  unit_of_analysis: year
  scale_unit: hectare

  revenues:
    - tourism: visitors * fee
    - premium: premium_revenue
    - carbon: area * sequestration * carbon_price
    - water: water_fee
    - contracts: contract_payments

  costs:
    - sfm: area * sfm_cost
    - labour: hours * wage
    - fixed: [energy, certification, marketing, admin, taxes]

  policy_dependence: 0.45

Each of the five BMs must have its own configuration reflecting
the Excel deterministic and Monte Carlo models.
